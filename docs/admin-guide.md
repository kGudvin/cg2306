# Инструкция администратора

## Доступ пользователей

Администратор управляет доступом через раздел `Админ-панель`.

Возможности:

- добавлять разрешенные email;
- назначать роль `manager` или `admin`;
- блокировать пользователей;
- видеть пользователей, которые уже входили в приложение.

Заблокированный пользователь не может войти. Его КП остаются доступными администратору до ручного или автоматического удаления.

## Шаблоны

В MVP по умолчанию создается один шаблон:

```text
КП ООО «Бештау Электроникс»
```

При первом старте приложение берет файл `КП от Кристины.docx`, создает из него подготовленную копию с плейсхолдерами и сохраняет ее как первую версию шаблона.

Через админ-панель можно загрузить новый DOCX-шаблон. Для production-шаблонов рекомендуется заранее расставлять плейсхолдеры вручную, чтобы не зависеть от автоматической подготовки.

## Хранение

- PostgreSQL хранит пользователей, КП, товары, суммы, шаблоны и аудит.
- DOCX/PDF файлы хранятся в Docker volume `cp_storage`.
- Хранится только последняя финальная версия DOCX/PDF.
- Автоматическое удаление КП через 100 календарных дней выполняется при старте приложения.

## Проверка после обновлений

После изменения кода или шаблона:

```powershell
docker compose up --build
```

Затем нужно заново сохранить КП и обновить предпросмотр, потому что старые PDF/DOCX не пересобираются автоматически.
