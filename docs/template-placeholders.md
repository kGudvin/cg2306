# DOCX-шаблон и плейсхолдеры

Документ генерируется из DOCX-шаблона. Приложение сохраняет связь КП с конкретной версией шаблона, чтобы старые КП не зависели от будущих обновлений шаблона.

## Поддерживаемые плейсхолдеры

Основные данные:

```text
{{outgoing_number}}
{{quote_date}}
{{recipient_name}}
{{recipient_inn}}
{{recipient_address}}
{{intro_text}}
{{specification_text}}
{{optional_conditions}}
```

Таблица товаров:

```text
{{item_no}}
{{item_name}}
{{item_unit}}
{{item_quantity}}
{{item_unit_price}}
{{item_line_total}}
```

Итоги:

```text
{{total_amount}}
{{vat_rate}}
{{vat_amount}}
{{total_amount_words}}
{{vat_amount_words}}
```

Условия:

```text
{{delivery_term}}
{{warranty}}
{{valid_until}}
```

Подписант:

```text
{{signer_title}}
{{signer_name}}
```

## Таблица товаров

Строка с плейсхолдерами `{{item_*}}` используется как образец строки товара. При генерации приложение:

1. копирует эту строку для каждой позиции;
2. сохраняет стиль строки;
3. удаляет исходную строку-плейсхолдер;
4. оставляет строку `ИТОГО` в шаблоне и подставляет итоговую сумму.

## Пустые блоки

Если `{{specification_text}}` или `{{optional_conditions}}` получают пустое значение, соответствующий абзац удаляется целиком.

## Важное для визуального совпадения

- Не меняйте структуру таблицы товаров без изменения генератора.
- Не разбивайте один плейсхолдер на несколько run-фрагментов в Word.
- Для стабильного PDF используйте шрифты, доступные LibreOffice в Docker-контейнере.
- После обновления шаблона старые КП остаются связаны со старой версией.
